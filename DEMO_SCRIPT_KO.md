# ReplyHub 3분 데모 촬영 대본

## 촬영 전 체크

- 휴대폰 방해 금지 모드 해제, 배터리 40% 이상, Wi-Fi 연결
- ReplyHub 알림 접근 허용 확인
- 설정에서 OpenAI API 키 연결 상태 확인
- 설정의 **데모 모드로 초기화** 실행
- 직접 전송 가능한 메신저의 테스트 발신 계정 준비
- 화면 녹화는 1080p, 터치 표시 켜기
- 비공개 연락처와 실제 알림은 모두 닫고 데모 계정만 사용

## 0:00-0:20 - 문제와 첫 화면

화면: ReplyHub AI 인박스

내레이션:

> 메신저가 다섯 개가 되면 답장은 대화가 아니라 앱 전환 작업이 됩니다. ReplyHub는 Android 알림을 사람별로 묶고, GPT-5.6으로 번역과 우선순위를 처리하는 하나의 AI 인박스입니다.

동작:

- 상단의 답장 필요, 긴급, 외국어 숫자를 짚는다.
- **답장 필요** 필터를 눌러 처리할 대화만 남긴다.

## 0:20-1:10 - 중국어 메시지와 GPT 웹 검색

화면: Li Wei 위챗 대화

내레이션:

> 중국어 원문과 한국어 번역이 함께 보입니다. 영업시간처럼 최신 사실이 필요한 질문은 웹 검색 답장을 선택합니다.

동작:

1. Li Wei 대화를 연다.
2. **답장**을 누른다.
3. **웹 검색**을 선택한다.
4. 지시란에 `오늘 영업시간을 확인해서 알려줘` 입력 또는 음성으로 말한다.
5. 초안을 생성한다.
6. 중국어 답장, 한국어 의미, AI 판단, 검색 근거와 출처를 차례로 보여준다.
7. 출처 링크 포함 여부를 보여주고 전송한다.

핵심 멘트:

> GPT가 단순히 문장을 예쁘게 만드는 것이 아니라 검색을 수행하고, 사용자가 확인할 수 있는 출처와 함께 상대 언어로 답합니다.

## 1:10-1:50 - 과거 대화 기억과 말투

화면: 민지 카카오톡 대화

동작:

1. `지난번에 말한 주소 다시 보내줄래?` 대화를 연다.
2. **맥락 답장**을 선택하고 초안을 만든다.
3. 과거 기록의 `강남대로 123, 2층`이 반영된 근거를 보여준다.
4. 존댓말 또는 반말 판단과 편집 가능한 초안을 보여준다.

내레이션:

> ReplyHub는 최근 메시지만 보는 것이 아니라 로컬 대화 기록에서 관련 내용을 찾고, 내가 전에 보낸 답장까지 포함해 관계의 말투를 유지합니다.

## 1:50-2:20 - 통합 연락처

화면: 임의의 대화 상세

동작:

1. Telegram의 Alex 대화에서 상단 연결 아이콘을 누른다.
2. WhatsApp의 Alex W. 대화를 선택한다.
3. 표시 이름을 정하고 연결한다.
4. 한 타임라인에 서로 다른 메신저 아이콘이 표시되는 장면을 보여준다.

내레이션:

> 사람은 앱이 아닙니다. 같은 사람의 Telegram과 WhatsApp 대화를 하나로 연결하고, 답장은 가장 최근 메시지가 온 채널로 이어집니다.

## 2:20-2:45 - 음성과 실제 전송

동작:

1. 설정에서 **데모 모드**를 끄고 실시간 알림 수집 상태를 보여준다.
2. 테스트 계정에서 메시지를 보내 활성 알림 대화를 만든다.
3. 해당 대화를 열고 마이크를 눌러 `오후 네 시 괜찮다고 답장해줘`라고 말한다.
4. 초안 생성 후 **바로 전송**을 누른다.
5. 상대 메신저에서 메시지가 전송된 장면을 확인한다.

내레이션:

> 활성 알림이 RemoteInput을 제공하면 다른 앱을 열지 않고 전송합니다. 지원하지 않으면 답장을 복사하고 정확한 메신저를 여는 안전한 폴백을 사용합니다.

## 2:45-3:00 - 마무리

화면: AI 인박스 전체

내레이션:

> ReplyHub는 모든 메신저를 위험하게 자동화하는 앱이 아닙니다. Android가 허용하는 범위 안에서 번역, 기억, 검색, 말투와 전송을 하나의 신뢰할 수 있는 대화 흐름으로 연결합니다.

자막:

`One AI inbox for every conversation, language, and messenger.`
